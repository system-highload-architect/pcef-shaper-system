# 📦 Reactive LRU Memory Chassis — Technical Specification (SRS)

### 📋 1. Core Objectives / Основные бизнес-цели
* **[RU]** Проектирование ультра-низкозатратного по утилизации CPU кэша памяти Use Case уровня для безопасного хранения, ленивой инвалидации и каскадного сжатия сессий абонентов в оперативной памяти Go кучи без использования тяжелых фоновых сканирующих горутин-дворников.
* **[EN]** Development of an ultra-low CPU overhead memory cache layer designed for secure tracking, lazy validation, and cascade eviction of user sessions inside the Go runtime heap, discarding heavy background scanning threads.

### ⚙️ 2. Algorithmic Logic & Tail-to-Head Cascade Eviction / Логика алгоритма
1. **Хронологическая шкала:** кэш инкапсулирует хэш-мапу за константное время $O(1)$ и двусвязный список `container/list` [🧠]. При каждом обращении (`Get`) узел абонента перемещается в голову списка (`MoveToFront`) [🧠]. Хвост списка (`Back()`) всегда нативно содержит самые старые и неактивные сессии [🧠].
2. **Ленивое вытеснение (Lazy Evaluation):** проверка `time.Since(entry.lastHeartbeat) > idleTimeout` выполняется строго реактивно в момент вызова `Get()`. Если сессия протухла — она удаляется, асинхронно вызывая финтех-колбэк возврата денег по Gy интерфейсу [🧠].
3. **Каскадное сжатие (Tail-to-Head Cascade):** при добавлении новой записи (`Set`), если размер кэша превышает жесткий лимит `capacity`, запускается цикл проверки с самого хвоста списка (`Back()`) [🧠]. Цикл последовательно вычищает протухшие элементы, стремясь сжать память к установленной границе, и **мгновенно останавливается**, как только натыкается на первую живую, активную запись, экономя такты CPU [🧠].
4. **Оптимизация RAM:** после массового удаления кэш принудительно инициирует неблокирующий системный вызов `runtime.GC()`, возвращая пустые страницы памяти хост-системе [🧠].

### 🎖 3. Acceptance Criteria / Критерии приемки кода
* Полный отказ от фоновых тикеров и таймеров на каждую сессию. Memory footprint удержания миллиона сессий оптимизирован до минимума [🧠].
* Тест конкурентности `go test -race` подтверждает полное отсутствие гонок данных под нагрузкой [🧠].
